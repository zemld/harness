## Scaffold project

Read:
- [stack.md](./stack.md) — every pinned library and rationale.
- [project-structure.md](./project-structure.md) — directory layout the scaffold must produce.
- [dependencies.md](./dependencies.md) — exact npm package list per concern.
- [deploy.md](./deploy.md) — Dockerfile and nginx config templates.

Conventions a fresh frontend project must satisfy:

- **Inputs the scaffold consumes:** project name (kebab-case), output directory, optional backend service name + backend OpenAPI path (used to configure the `gen:api` script for the project's `package.json`).
- **Top-level layout** (per [project-structure.md](./project-structure.md)): `src/{app,pages,features,components/ui,lib,api,styles,mocks}/`, `tests/e2e/`, `deploy/`, `public/`.
- **Required boilerplate under `src/`:**
  - `app/{App.tsx,providers.tsx,router.tsx}` — composition root (QueryClientProvider, RouterProvider, theme; data-router route table per [routing.md](./routing.md)).
  - `api/{client.ts,generated.ts}` — configured `openapi-fetch` instance and the codegen placeholder (regenerated by `gen:api`, never hand-edited).
  - `styles/globals.css` — Tailwind v4 entry + CSS variables.
  - `mocks/handlers.ts` — placeholder MSW handlers.
  - `main.tsx` — Vite entry that mounts `App` and imports `styles/globals.css`.
- **Required boilerplate under `tests/`:** `setup.ts` (loads `@testing-library/jest-dom`, starts the MSW server using handlers from `src/mocks/handlers.ts`).
- **Required boilerplate under `deploy/`:** `Dockerfile` and `nginx.conf` matching [deploy.md](./deploy.md) verbatim.
- **Required boilerplate under `public/`:** `config.js` placeholder for runtime `window.__APP_CONFIG__` per [deploy.md](./deploy.md)'s build-time-vs-runtime configuration pattern.
- **Required config files at the project root** (per [stack.md](./stack.md) and [dependencies.md](./dependencies.md)): `package.json` (with the pinned dependency set and the standard scripts — `dev`, `build`, `preview`, `test`, `test:watch`, `e2e`, `format`, `lint`, `typecheck`, `gen:api`, `prepare`), `tsconfig.json` (must enable `strict`, `noUncheckedIndexedAccess`, `verbatimModuleSyntax`, `noUnusedLocals`, `noUnusedParameters`, plus `@/* → src/*` path alias), `vite.config.ts` (React + Tailwind v4 plugins, `@ → src` alias, Vitest config with `happy-dom` and the `tests/setup.ts` setup file), `biome.json` (recommended rules, double quotes, semicolons, 2-space indent, 100-char width), `lefthook.yml` (pre-commit hooks for `biome check --staged` + `tsc --noEmit`). pnpm is the package manager — `pnpm-lock.yaml` is committed; no `package-lock.json` / `yarn.lock` may exist.
- **Post-scaffold hand-off conventions:** the runner must leave the project in a state where `tsc --noEmit` and `biome check .` both pass. Lefthook hooks must be installed (`lefthook install`) so the first commit runs the pre-commit pipeline. The user is then responsible for configuring `gen:api` if the backend OpenAPI path was not known at scaffold time, bringing in shadcn primitives via the shadcn CLI, adding the first real route under `src/pages/`, implementing MSW handlers for the first feature, and adding infra entries (per [deploy.md](./deploy.md)) when the project joins an existing infra layout.

The runner that produces this layout — exact pnpm commands, file contents, etc. — is the responsibility of the scaffolding skill; this index does not prescribe its implementation.

## Refactor project

Read every rule doc in this directory: [stack.md](./stack.md), [project-structure.md](./project-structure.md), [style.md](./style.md), [state.md](./state.md), [forms.md](./forms.md), [api-integration.md](./api-integration.md), [routing.md](./routing.md), [testing.md](./testing.md), [dependencies.md](./dependencies.md), [deploy.md](./deploy.md).

Audit the entire project (every `.ts` / `.tsx` / `.css` file under `src/` and `tests/`, plus `package.json`, `tsconfig.json`, `vite.config.ts`, `biome.json`, `lefthook.yml`, `deploy/Dockerfile`, `deploy/nginx.conf`). For each violation record one row in the relevant category table.

Categories:
- **Structure** — [project-structure.md](./project-structure.md).
- **State** — [state.md](./state.md).
- **Forms** — [forms.md](./forms.md).
- **API** — [api-integration.md](./api-integration.md).
- **Routing** — [routing.md](./routing.md).
- **Testing** — [testing.md](./testing.md).
- **Style** — [style.md](./style.md).
- **Tooling** — [stack.md](./stack.md) + [dependencies.md](./dependencies.md) (presence and shape of `biome.json`, `lefthook.yml`, `tsconfig.json` strict flags, `package.json` library pins, `pnpm-lock.yaml` committed and no `package-lock.json` / `yarn.lock`).

Per-violation row: `File:Line | Rule | Violation | Complexity (direct | behavioral)`.

Direct fixes (apply in-session): move files into `features/<feature>/`, delete stray barrel `index.ts`, relocate `utils/` content into `lib/`, move E2E specs into `tests/e2e/`, split overgrown pages by extracting feature components, replace raw `fetch`/`axios` with `apiClient`, replace hand-written payload types with imports from `src/api/generated.ts`, relocate query hooks to `features/<feature>/api.ts`, convert JSX `<Route>` to route-object form, remove `React.FC`, fix prop typing, add missing `biome.json` / `lefthook.yml`, patch `tsconfig.json` strict flags, remove `package-lock.json` / `yarn.lock`.

Behavioral fixes (carve out as a scoped follow-up feature and run through the standard feature pipeline — design, tests, implementation, verification):
- Form uses `useState` instead of RHF + Zod.
- Component duplicates server state into Zustand.
- Tests query by `getByTestId` instead of by role.
- Feature has no tests.

After all direct fixes: `pnpm biome check --write . && pnpm tsc --noEmit`.
